Java Minesweeper

by: Sam Kahn

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I wanted to make a game that would be challenging and original. I thought that others might not have created a minesweeper game for level 2, so I went with it. My original thought was to make a platformer, but that seemed too ambitious for my level of coding experience. I was playing a lot of minesweeper at the time, so it seemed like a good game that would be at just the right level of difficulty.

2. What I Learned

I learned that my algorithms were broken. Some of the tiles simply came up with errors when clicked on. I had to rework my logic.

3. Architecture of My Game

My game includes tiles, bombs, and flags. The tiles themselves can be blank, numbered, or bombs. The numbered tiles indicate the amount of bombs around them. Blank tiles indicate that all eight tiles around it have 0 bombs. The flags are the win condition. Every bomb must be flagged for a player to win. The structure involves an overarching class, a GamePanel, a Tile object, and a TileManager. The tiles are managed by the TileManager, and the GamePanel deals with the actual creation of the graphics. The overarching class is there to make the frame.

4. Technical Implementation

The most complicated part of my game was the logic involved in detecting bombs in surrounding tiles. I had to use a mouse listener to identify the clicked tile, and then detect whether each tile around it was a bomb or not. This was difficult, because the grid was lined up in order by line. So a tile on one line could detect a tile on another separate line. This led to cross-board detection, which I quickly fixed. Edge tiles also caused errors because it couldn't detect what didn't exist. I started by making the logic, and then from there I troubleshooted for a few weeks. It took a while to get it to work perfectly.

5. Most Challenging Part

The most challenging part was also the logic. Tiles were causing errors because it couldn't find other tiles, and numbers were showing up wrong due to them detecting the wrong tiles. I overcame this challenge through a lot of testing and perseverance.

6. Future Fixes and Updates

I would like to add a scoreboard so the player can see how many bombs are left. I would also like to fix a bug found in testing where the player could simply flag every tile and win. This was due to my win condition only detecting flags. I didn't realize till I had already exported the game.
